\begin{abstract}
  Certifying lower bounds for Ramsey numbers is a challenging discrete optimization problem over edge colorings of complete graphs. We formulate Ramsey search as continuous optimization over probabilistic edge colorings, replace the raw expected monochromatic-clique count by a log-barrier surrogate that emphasizes near-violations, and optimize the resulting objective with a Frank-Wolfe type method and stochastic clique batching. The resulting pipeline is simple, projection-free, and effective on diagonal and near-diagonal instances. In particular, it repeatedly recovers $42$-vertex witnesses for the lower bound on $R(5,5)$ and produces a large collection of feasible colorings in that regime. These results suggest that first-order continuous optimization can be competitive for certified combinatorial search problems.
\end{abstract}



\section{Introduction}

Ramsey theory studies the emergence of unavoidable structure in sufficiently large graphs. A central problem is to certify lower bounds for Ramsey numbers by exhibiting edge colorings of a complete graph that avoid forbidden monochromatic cliques. This viewpoint leads naturally to a discrete optimization problem over graph colorings and provides a clean testbed for continuous methods on combinatorial objects.

Let $G_n=(V,E)$ be the complete graph on $n$ vertices, and write $m=\binom{n}{2}$ for the number of edges. A binary coloring is represented by a vector $x=(x_{ij})_{1 \leq i < j \leq n} \in \{0,1\}^m$, where $x_{ij}=1$ denotes that edge $(i,j)$ receives the first color and $x_{ij}=0$ the second. For integers $r,s \geq 2$, the Ramsey number $R(r,s)$ is the smallest $n$ such that every coloring of $G_n$ contains either a monochromatic $K_r$ in color $1$ or a monochromatic $K_s$ in color $0$. Consequently, a certificate for the lower bound $R(r,s)>n$ is exactly a coloring of $G_n$ that avoids both structures.

This search can be written as the minimization of the number of forbidden monochromatic cliques. Let $\mathcal{C}_k$ denote the set of $k$-vertex subsets of $V$. Then the objective

\konrad{Not super happy with the notation, one could also be more explicit that one just optimizes over $\{0,1\}^{\binom{n}{2}}$.}

\begin{equation}\label{eq:no-monochromatic-cliques}
    M(x) = \sum_{C \in \mathcal{C}_r} \prod_{\{i,j\} \subseteq C} x_{ij}
    \quad + \quad
    \sum_{C \in \mathcal{C}_s} \prod_{\{i,j\} \subseteq C} \left(1 - x_{ij}\right)
\end{equation}

counts monochromatic $K_r$ subgraphs in color $1$ together with monochromatic $K_s$ subgraphs in color $0$. A zero-valued solution therefore gives a valid witness for $R(r,s)>n$. The corresponding discrete optimization problem is

\begin{equation}\label{eq:opt-problem}
    \min_{x \in \{0,1\}^m} M(x).
\end{equation}

Recent work has highlighted the value of computational search for extremal combinatorics, ranging from classical heuristic search to neural and LLM-assisted pipelines \cite{Rubinstein2004,wagner2021constructions,nagda2026reinforcedgenerationcombinatorialstructures}. In contrast to highly specialized search procedures, we study a single continuous optimization pipeline built directly from the clique-counting objective. Ramsey search is an attractive benchmark for this purpose: the relaxation is simple, the objective is highly nonconvex and high-order, and success is measured by explicit, verifiable witnesses. We refer to \cite{morris2026recentresultsramseytheory} for a recent overview of Ramsey-theoretic background.

\paragraph{Contributions.}
\begin{itemize}
  \item We formulate Ramsey search as continuous optimization over probabilistic edge colorings of complete graphs.
  \item We introduce a log-barrier surrogate that preserves the clique-wise structure of the problem while providing a stronger optimization signal than the raw expected clique count.
  \item We apply a Frank-Wolfe type method to the relaxed problem, yielding projection-free updates that naturally move iterates toward discrete colorings \cite{frank1956algorithm,braun2022conditional}.
  \item We develop stochastic clique batching schemes that make optimization feasible when exact evaluation over all cliques is too expensive.
  \item Empirically, the method recovers strong witnesses on diagonal and near-diagonal instances, including repeated recovery of $42$-vertex constructions for the lower bound on $R(5,5)$.
\end{itemize}


\section{Continuous Relaxation}

The discrete feasible set in \cref{eq:opt-problem} admits an immediate continuous relaxation:

\begin{equation}\label{eq:relaxed-problem}
    \min_{x \in [0,1]^m} M(x).
\end{equation}

Under this relaxation, each variable $x_{ij}$ can be interpreted as the probability that edge $(i,j)$ receives color $1$ under an independent Bernoulli coloring model. The objective $M(x)$ is then the expected number of forbidden monochromatic cliques, since each product term is exactly the probability that a given clique is monochromatic in one of the two colors. This probabilistic view preserves a direct connection to the discrete search problem while exposing a continuous domain on which first-order methods can operate.

Equivalently, the relaxed problem can be written as

\begin{align}\label{eq:constrained-problem}
    \min_{x} \quad &\sum_{C \in \mathcal{C}_r} \prod_{\{i,j\} \subseteq C} x_{ij}
    \quad + \quad
    \sum_{C \in \mathcal{C}_s} \prod_{\{i,j\} \subseteq C} \left(1 - x_{ij}\right), \\
    \text{s.t.} \quad &x \in [0,1]^m.
\end{align}

The objective is a polynomial of degree $\binom{\max(r,s)}{2}$ in $m=\binom{n}{2}$ variables, so gradients can be computed explicitly whenever clique enumeration is feasible. Although any final certificate must still be discrete, the continuous relaxation is useful because it exposes informative descent directions and allows the optimizer to exploit partial progress before a valid witness has fully polarized.

Ultimately, lower bounds are certified only after converting a candidate point to a binary coloring and verifying directly that no forbidden monochromatic cliques remain.

In our implementation, this extraction is performed deterministically by the method of conditional expectation rather than by thresholding or randomized rounding. Starting from a relaxed iterate $x$, we process the edges one at a time. For the current undecided edge $e$, we evaluate the objective conditioned on fixing $e=0$ and on fixing $e=1$, while keeping the remaining undecided edges at their current fractional values. We then permanently assign the value that yields the smaller conditional objective. Repeating this over all edges produces a binary coloring whose final number of monochromatic clique violations is no larger than the conditional expectation encoded by the relaxed point. The resulting coloring is then verified directly by exhaustive clique checking (or simply running a max clique algorithm); if no forbidden monochromatic $K_r$ or $K_s$ is found, it certifies the claimed Ramsey lower bound.

\section{Objective Design}

The relaxed objective in \cref{eq:relaxed-problem} is conceptually natural, but in practice it provides a weak optimization signal. Because each clique contribution is a high-order product of numbers in $[0,1]$, the raw objective often yields very small gradients even when a candidate solution is close to creating a forbidden monochromatic clique. As a result, a first-order method can spend many iterations reducing already-safe regions of the graph while paying insufficient attention to the most dangerous near-violations.

To make this issue explicit, consider the raw expected-count objective
\begin{equation}\label{eq:linear_expectation}
    M(x)
    =
    \sum_{C \in \mathcal{C}_r} \prod_{\{i,j\} \subseteq C} x_{ij}
    +
    \sum_{C \in \mathcal{C}_s} \prod_{\{i,j\} \subseteq C} (1 - x_{ij}).
\end{equation}
When a clique differs from a monochromatic one by only a small number of edges, the corresponding product term can still be numerically tiny, especially for larger clique sizes. The optimizer is then encouraged to distribute mass diffusely rather than to eliminate the specific substructures most likely to invalidate a witness.

We therefore optimize a barrier-style surrogate that penalizes each candidate forbidden clique individually:
\begin{equation}\label{eq:log_barrier}
    L(x)
    =
    -\sum_{C \in \mathcal{C}_r}
    \log\left(1 - \prod_{\{i,j\} \subseteq C} x_{ij}\right)
    -
    \sum_{C \in \mathcal{C}_s}
    \log\left(1 - \prod_{\{i,j\} \subseteq C} (1 - x_{ij})\right).
\end{equation}
The logarithmic barrier changes the geometry of the problem in a useful way. Each clique behaves like an individual soft constraint, and the penalty for a nearly monochromatic clique grows rapidly as its monochromatic probability approaches $1$. Consequently, gradients concentrate on the substructures most likely to violate the target Ramsey condition.

\section{Frank-Wolfe Optimization}

We optimize the relaxed objective over the hypercube $[0,1]^m$ with a Frank-Wolfe type method \cite{frank1956algorithm,braun2022conditional}. At iteration $t$, given a current iterate $x_t$, the linear minimization subproblem is
\begin{equation}
    s_t \in \arg\min_{s \in [0,1]^m} \langle \nabla L(x_t), s \rangle.
\end{equation}
Because the feasible region is a hypercube, this oracle has a simple closed form:
\begin{equation}
    (s_t)_{ij} =
    \begin{cases}
        0, & \text{if } \frac{\partial L(x_t)}{\partial x_{ij}} > 0, \\
        1, & \text{otherwise.}
    \end{cases}
\end{equation}


In the actual implementation, even the two-color case is handled in a multicolor parameterization: for each edge $\{i,j\}$ we optimize a probability vector $x_{ij} \in \Delta^{c-1}$ over colors rather than a single scalar in $[0,1]$. The Frank-Wolfe linear minimization oracle remains equally simple. Writing $g_{ij} \in \mathbb{R}^c$ for the gradient with respect to the color probabilities on edge $\{i,j\}$, the oracle solves
\[
(s_t)_{ij} \in \arg\min_{z \in \Delta^{c-1}} \langle g_{ij}, z \rangle,
\]
and therefore places all mass on a minimizing color:
\[
(s_t)_{ij} = e_{k^*},
\qquad
k^* \in \arg\min_{k \in \{1,\dots,c\}} (g_{ij})_k,
\]
where $e_{k^*}$ denotes the corresponding standard basis vector. Thus, exactly as in the hypercube formulation, the oracle returns an extreme point, now interpreted as choosing one color for each edge.

The iterate is then updated by
\begin{equation}
    x_{t+1} = (1-\gamma_t)x_t + \gamma_t s_t,
\end{equation}
where $\gamma_t \in [0,1]$ is a step size.

This choice is attractive for two reasons. First, each update is projection-free and inexpensive even in very high dimension. Second, the linear minimization oracle always returns a vertex of the hypercube, so the iterates are repeatedly pulled toward discrete colorings rather than remaining in the interior. In our setting this bias toward extreme points is desirable because the final goal is a binary witness, not merely a low-value continuous relaxation.

\section{Scalable Batched Optimization}

The main computational bottleneck is clique enumeration. Even for moderate $n$, the number of $k$-cliques,
\[
|\mathcal{C}_k| = \binom{n}{k},
\]
is far too large for exact evaluation at every Frank-Wolfe step. Since our objectives are sums of independent clique-wise penalties, the natural approximation is to replace the full sum by a mini-batch average.

Let $\ell_r(C;x)$ and $\ell_s(C;x)$ denote the contribution of clique $C$ to the two color classes. For the barrier surrogate in \cref{eq:log_barrier},
\[
\ell_r(C;x) = -\log\left(1 - \prod_{\{i,j\} \subseteq C} x_{ij}\right),
\qquad
\ell_s(C;x) = -\log\left(1 - \prod_{\{i,j\} \subseteq C} (1-x_{ij})\right).
\]
For batching, it is convenient to work with the clique-averaged objective
\[
\bar{L}(x)
=
\frac{1}{|\mathcal{C}_r|}
\sum_{C \in \mathcal{C}_r} \ell_r(C;x)
+
\frac{1}{|\mathcal{C}_s|}
\sum_{C \in \mathcal{C}_s} \ell_s(C;x),
\]
whose gradient has the same form as the original objective up to a constant rescaling.

The diagonal case $r=s$ is straightforward. Both color terms are indexed by the same family of $r$-cliques, so we sample a single batch $S \subseteq \mathcal{C}_r$ and evaluate both colors on that same set:
\[
\widehat{L}_{\mathrm{diag}}(x)
=
\frac{1}{|S|}
\sum_{C \in S} \bigl(\ell_r(C;x) + \ell_s(C;x)\bigr).
\]
This is an unbiased estimator of $\bar{L}(x)$. Using the same batch for both colors is also the most intuitive choice: each stochastic update compares the red and blue penalties on exactly the same local vertex pattern, rather than adding variance from two unrelated samples.

The off-diagonal case $r \neq s$ requires one more design choice because the two color terms live on different clique families. Here the main question is not whether to batch, but how to organize the two batches across an epoch.

Let
\[
R = \max(r,s), \qquad q = \min(r,s),
\]
and let $\ell_R$ and $\ell_q$ denote the clique penalties for the corresponding two colors. We choose a batch size $B_R$ for the larger clique family $\mathcal{C}_R$, and define
\[
T = \left\lceil \frac{|\mathcal{C}_R|}{B_R} \right\rceil
\]
to be the number of optimization steps in one epoch. At the start of the epoch, we randomly permute all $R$-cliques and partition them into batches
\[
A_1,\dots,A_T,
\qquad |A_t| \leq B_R.
\]
We then independently permute the smaller clique family $\mathcal{C}_q$ and partition it into the same number $T$ of batches
\[
B_1,\dots,B_T,
\qquad |B_t| \approx \frac{|\mathcal{C}_q|}{T}.
\]
Equivalently, the smaller-clique batch size is chosen so that both colors finish one full pass through their respective clique families at the same time.

At optimization step $t$, we evaluate one batch from each color and then apply a single update:
\[
\widehat{\mathcal{L}}_t(x)
=
\frac{1}{|A_t|}
\sum_{C \in A_t} \ell_R(C;x)
\;+\;
\frac{1}{|B_t|}
\sum_{D \in B_t} \ell_q(D;x).
\]
This is what ``independent batching'' means in our epoch-based implementation. The two color classes are sampled independently, but they are synchronized at the level of epochs: each step uses the $t$-th batch from the large-clique pass and the $t$-th batch from the small-clique pass, and after $T$ steps both passes are complete.

This viewpoint is often clearer than thinking of each update as drawing two fresh random mini-batches from scratch. Over one epoch, every clique in each family is visited exactly once, up to the usual rounding in the last batch, and reshuffling happens only when the next epoch begins. The resulting estimator is still unbiased for the same clique-averaged objective, while fitting naturally with the ``one pass through all cliques per epoch'' interpretation used in the implementation.

\section{Results}

Cases that work so far (i.e., where we match the known lower bound (or exact value)):

\begin{itemize}
  \item $R(3,k)$ for $k\leq 8$: easy. $R(3, 9)$ should also be doable.
  \item $R(4,4)$ and $R(4,5)$: easy as well.
  \item $R(5,5)$: found $\approx 100$ distinct $42$-colorings, hopes are that there will be more (all 328 seems a bit unrealistic).
\end{itemize}

What doesn't work (yet):

\begin{itemize}
    \item $R(4, 6)$: more difficult than expected, still haven't found one, current "record" are 5 monochromatic cliques on 35 nodes.
    \item $R(4, 7)$: started some runs but here the runs are quite expensive already.
    \item $R(5, 6)$: same.
\end{itemize}

We evaluate the proposed method on diagonal and near-diagonal Ramsey instances. The headline result is the repeated recovery of feasible $42$-vertex colorings for the lower bound on $R(5,5)$, matching the strongest known construction at that size. Beyond a single witness, the method produces a large collection of successful solutions, which indicates that the optimization landscape contains a nontrivial family of high-quality constructions rather than a single isolated optimum.
